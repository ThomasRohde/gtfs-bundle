# Summary

GTFS.org maintains a resources area that describes a community list of standards, APIs, apps, tools, datasets, and research around open public-transit technology. It frames open technology as including open standards, open data, and open-source software such as OpenTripPlanner and the GTFS Validator. [1]

Use this as a discovery hub when a task needs tooling beyond this bundle: producing feeds, validating feeds, using data, finding APIs, or comparing ecosystem options. Treat it as an implementation-resource index, not as normative GTFS Schedule requirements. [1]

# Caveats

- Tools listed in community resources may have their own licenses, data assumptions, and support status.
- Community tools can implement only a subset of current GTFS; check tool documentation before assuming support for Fares v2, Flex, pathways, or Realtime.
